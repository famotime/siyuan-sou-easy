# 思源笔记插件开发计划：更友好的查找与替换

## 1. 开发目标

基于 `plugin-sample-vite-vue` 模板，在当前仓库内先完成可运行的 V1：当前文档内的 IDE 风格逐条查找替换。

## 2. 技术路线

### 2.1 总体架构

拆成五层：

- 插件入口层：注册命令、顶栏入口、生命周期、事件监听；
- 编辑器上下文层：识别当前激活文档与当前 protyle；
- 搜索引擎层：从文档块构建可搜索结果集并计算命中；
- 替换执行层：将单个命中映射回块并调用官方 API 更新；
- UI 展示层：悬浮查找替换组件、高亮与反馈。

### 2.2 推荐目录规划

建议在现有 `src` 下新增：

- `src/plugin/`：插件生命周期、命令、事件绑定；
- `src/features/search-replace/`：核心业务；
- `src/features/search-replace/model.ts`：结果模型与类型；
- `src/features/search-replace/search-engine.ts`：匹配计算；
- `src/features/search-replace/replace-engine.ts`：替换执行；
- `src/features/search-replace/context.ts`：当前编辑器上下文；
- `src/components/search-replace/`：查找替换 UI；
- `src/stores/`：面板状态；
- `src/utils/`：通用 API 封装与字符串工具。

## 3. 分阶段计划

## 阶段 0：项目初始化与模板清理

### 目标

把样板工程改造成真实插件骨架。

### 任务

- 修改 `plugin.json`、`package.json`、`README.md` 中的样板名称；
- 确定插件 ID、显示名、中英文描述；
- 删除或替换 `App.vue` 中的示例逻辑；
- 保留并复用现有思源风格组件；
- 建立 `requestApi` 统一请求封装；
- 建立基础 i18n 文案键。

### 交付物

- 可正常加载的插件骨架；
- 统一 API 调用层；
- 清晰的目录结构。

## 阶段 1：编辑器上下文识别

### 目标

可靠获取“当前激活文档”和当前编辑器上下文。

### 任务

- 监听 `switch-protyle`、`loaded-protyle-static`、`destroy-protyle`；
- 维护当前活动文档的 `rootId`、`blockId`、DOM 容器引用；
- 设计上下文失效与重建机制；
- 在多标签 / 分栏切换时保证上下文同步。

### 验收

- 文档切换后，插件知道当前操作目标；
- 关闭文档或切换编辑器时，旧状态被正确清理。

## 阶段 2：文档索引与搜索引擎

### 目标

完成当前文档的块索引、搜索规则解析和命中列表计算。

### 任务

- 使用 `/api/query/sql` 按 `root_id` 拉取当前文档块；
- 建立块顺序与可搜索块白名单；
- 实现普通文本、大小写、全词、正则四类匹配逻辑；
- 输出标准化结果结构：块、偏移、预览、是否可替换；
- 对非法正则进行错误处理；
- 对输入增加防抖与缓存。

### 建议实现细节

- 先以 `content` 做候选筛选；
- 需要精确替换时再取 `markdown` / `kramdown`；
- 全词匹配使用边界判断，不直接依赖简单空格分词；
- 正则模式下单独做异常捕获。

### 验收

- 输入任意查询词后可以得到稳定、可排序的命中列表；
- 模式切换后结果即时刷新；
- 大文档下不会明显阻塞 UI。

## 阶段 3：悬浮查找替换 UI

### 目标

做出贴近 VS Code 的交互面板。

### 任务

- 实现悬浮查找条与折叠式替换条；
- 增加结果计数、上一项 / 下一项、模式切换按钮；
- 支持键盘焦点切换与回车导航；
- 支持关闭面板与重新打开时恢复上次状态；
- 使用思源主题变量适配暗色 / 亮色主题。

### 验收

- UI 可在当前编辑区上方稳定显示；
- 用户可以只用键盘完成主要流程；
- 当前结果索引与界面状态一致。

## 阶段 4：命中定位与高亮

### 目标

让用户明确看到“当前处理的是哪一处文本”。

### 任务

- 将当前命中映射到编辑区 DOM；
- 自动滚动到当前命中附近；
- 绘制当前高亮与普通高亮；
- 处理切换命中时的高亮清理；
- 在文档刷新后尝试恢复当前命中定位。

### 风险点

- 思源 DOM 结构随版本可能变化；
- 部分复杂块的文本节点映射困难。

### 策略

- V1 优先保证“当前命中定位”稳定；
- 普通高亮可先做简化版或延后。

## 阶段 5：替换执行引擎

### 目标

完成逐条替换、跳过、全部替换的安全执行。

### 任务

- 拉取目标块原始 `kramdown` 或 `markdown`；
- 建立“命中偏移 → 块内容更新”的映射逻辑；
- 调用 `/api/block/updateBlock` 更新块内容；
- 替换后刷新当前结果集并定位下一项；
- 对全部替换增加确认与汇总反馈；
- 对块已变化、命中失效等情况做回退处理。

### 验收

- 替换当前只影响一个命中；
- 跳过当前不修改任何数据；
- 全部替换后有准确统计。

## 阶段 6：设置页与可配置项

### 目标

补齐基础可配置能力。

### 任务

- 增加设置页；
- 支持是否搜索代码块；
- 支持默认展开替换栏；
- 支持调试日志；
- 为后续“保留大小写”预留开关。

## 阶段 7：测试与发布

### 目标

确保 V1 可回归、可打包、可安装。

### 测试策略

- 单元测试：
  - 搜索规则解析；
  - 全词匹配边界；
  - 正则异常处理；
  - 替换偏移映射；
- 手工集成测试：
  - 当前文档搜索；
  - 连续替换与跳过；
  - 大文档性能；
  - 中英文混排；
  - 正则替换；
  - 复杂块降级行为。

### 发布前检查

- 检查 `plugin.json` 的版本号与 `minAppVersion`；
- 检查 i18n 文案完整性；
- 检查深浅主题显示；
- 构建 `package.zip` 并在思源中手工验证安装。

## 4. 任务拆解清单

建议按如下顺序执行：

1. 清理模板元数据与入口；
2. 提取统一 API 请求层；
3. 实现当前文档上下文识别；
4. 实现文档块查询与结果模型；
5. 实现搜索引擎与模式切换；
6. 实现悬浮 UI；
7. 实现当前命中导航；
8. 实现当前命中高亮；
9. 实现替换当前与跳过；
10. 实现全部替换与汇总；
11. 增加设置页；
12. 补测试与发布文档。

## 5. 关键技术决策

### 决策 1：V1 仅支持当前文档

原因：最贴近 VS Code 编辑器内查找体验，且风险最低。

### 决策 2：SQL 仅做检索，写操作统一走 API

原因：符合思源推荐边界，更安全。

### 决策 3：V1 不支持跨块命中替换

原因：底层数据模型与用户心智不一致，强行支持风险高且收益有限。

### 决策 4：复杂块采用白名单支持

原因：先保证常用文本块稳定，再逐步扩展。

## 6. 风险排期建议

- 第一优先级风险：当前命中定位与替换映射；
- 第二优先级风险：编辑器切换状态同步；
- 第三优先级风险：长文档性能；
- 第四优先级风险：复杂块兼容。

建议先完成“当前文档 + 普通文本块 + 单条替换”的最小闭环，再扩展高亮、设置与增强能力。

## 7. 里程碑建议

- M1：插件骨架可运行，能识别当前文档；
- M2：可以搜索并得到命中列表；
- M3：可以导航到当前命中；
- M4：可以替换当前 / 跳过当前；
- M5：可以全部替换并发布首个可用版本。
