# 手工验证清单

用于在思源笔记桌面端手工验证 `siyuan-friendly-search-replace` 当前版本的核心功能。

## 1. 验证前准备

- 执行 `npm install`
- 执行 `npm run test`
- 执行 `npm run precheck`
- 执行 `npm run build`
- 将生成的插件包或 `dist` 内容安装到思源插件目录
- 打开一个包含普通段落、标题、列表、代码块、复杂富文本的测试文档

## 2. 基础入口

- 顶栏按钮可以打开查找面板
- 命令面板可以打开查找面板和替换面板
- 自定义快捷键修改后立即生效
- 面板关闭后再次打开，状态符合当前设置

## 3. 查找能力

- 普通文本查找结果数量正确
- `区分大小写` 开关会立即影响结果
- `全词匹配` 可以过滤 `foobar` 这类非整词命中
- `正则匹配` 可以正确返回结果
- 非法正则会提示错误，而不是导致面板卡死

## 4. 导航与高亮

- 上一项 / 下一项导航顺序正确
- 当前命中切换时会滚动到可见区域
- 当前命中和普通命中都有明显视觉区分
- 切换文档后结果会刷新，不会继续停留在旧文档状态

## 5. 替换能力

- `替换当前` 只修改当前命中
- `跳过` 不修改内容，只切到下一项
- `全部替换` 会弹出确认，并显示替换统计
- 复杂富文本命中在不可安全替换时会降级提示

## 6. 保留大小写

使用查找词 `foo`、替换词 `bar`，分别验证：

- 命中 `foo` 时替换结果为 `bar`
- 命中 `FOO` 时替换结果为 `BAR`
- 命中 `Foo` 或 `Old Value` 时替换结果为 `Bar` 或 `New Value`
- 命中混合大小写（如 `fOo`）时，保持原始替换文本不变

## 7. 设置项

- `默认展开替换栏` 生效
- `记住面板位置` 生效，拖动后重开仍能恢复
- `优先使用选中文本` 生效
- `搜索代码块` 打开和关闭后结果有差异
- `调试日志` 打开后，控制台可看到 `[sfsr]` 日志
- `保留大小写` 打开和关闭后替换结果有差异

## 8. 发布前复核

- `plugin.json` 版本号正确
- 中英文 README 与 i18n 文案同步
- 深色 / 浅色主题显示正常
- `package.zip` 可被思源正常安装
