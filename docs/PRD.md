# 思源笔记插件 PRD：更友好的查找与替换

> 工作名：`siyuan-friendly-search-replace`
>
> 说明：用户提到的模板路径为 `/plugin-sample-vite-vue-main`，当前仓库实际可用模板目录为 `plugin-sample-vite-vue`；用户提到的 `/reference` 在本仓库对应为 `reference_docs`。

## 1. 项目背景

思源笔记当前的查找与替换更偏向“按块处理”的工作方式，这与用户在文本编辑器、IDE 中习惯的“按文本命中逐个跳转、逐个替换、可跳过”的体验存在明显差异。

目标是基于思源现有块模型与公开 API，构建一个更接近 VS Code 编辑器内查找替换体验的插件：

- 在当前文档内按文本命中顺序逐个定位；
- 支持上一项 / 下一项导航；
- 支持替换当前命中、跳过当前命中、批量全部替换；
- 让用户感觉自己在操作“文本结果”，而不是“块结果”。

## 3. VS Code 查找替换界面功能总结

基于截图与 VS Code 编辑器查找组件的典型交互，可抽象出以下核心能力：

### 3.1 查找输入区

- 输入查找关键词；
- 即时显示命中计数，如 `3 of 5`；
- 支持上一项 / 下一项切换当前命中；
- 支持关闭查找条。

### 3.2 查找模式开关

- 区分大小写（Match Case）；
- 全词匹配（Match Whole Word）；
- 正则匹配（Use Regular Expression）。

### 3.3 替换输入区

- 支持展开 / 折叠替换栏；
- 输入替换内容；
- 支持替换当前命中；
- 支持全部替换；
- 存在“保留大小写”一类的高级替换选项入口。

### 3.4 交互特征

- 操作对象是“当前文本命中”，不是块或整段；
- 结果导航与替换行为紧密耦合；
- 当前命中高亮明确，且通常会自动滚动到可见位置；
- 面板轻量、悬浮、不打断主编辑流程。

## 4. 产品定位

这是一个面向思源桌面端编辑场景的增强插件，用 IDE 风格的交互包装思源底层块能力，让用户以“逐个文本命中”的方式完成当前文档内的查找与替换。

## 5. 目标与非目标

### 5.1 V1 目标

- 在当前激活文档内提供类似 VS Code 的悬浮查找替换面板；
- 支持按文本命中逐个定位、逐个替换、跳过；
- 支持大小写、全词、正则三种匹配模式；
- 支持替换当前与全部替换；
- 基于思源 API 安全落地，不直接写数据库。

### 5.2 V1 非目标

- 不做全工作空间级别的搜索替换；
- 不支持“跨多个块的单次命中替换”；
- 不直接改写数据库表；
- 不承诺覆盖所有复杂块类型的无损替换。

## 6. 目标用户

- 从 VS Code、JetBrains、Typora 等工具迁移到思源的重度编辑用户；
- 经常需要在长文档中逐条确认替换结果的用户；
- 需要“跳过某个命中，但继续处理下一个命中”的用户。


## 8. 范围定义

### 8.1 搜索范围

V1 仅支持：当前激活文档。

说明：该范围最接近 VS Code 编辑器内查找条，也最适合思源当前块模型下的安全实现。

### 8.2 匹配单位

- 用户感知单位：文本命中；
- 系统实现单位：文档内各文本块中的命中位置；
- 命中顺序：按文档块顺序 + 块内字符偏移顺序排序。

### 8.3 替换边界

- 允许：单个块内部的文本替换；
- 不允许：一个命中跨越多个块；
- 对复杂块类型可降级为“只搜索不替换”或“暂不支持”。

## 9. 功能需求

## 9.2 查找面板

面板需包含：

- 查找输入框；
- 匹配结果计数；
- 上一项 / 下一项按钮；
- 区分大小写开关；
- 全词匹配开关；
- 正则匹配开关；
- 关闭按钮；
- 更多操作菜单预留位。

交互要求：

- 输入后自动刷新结果；
- 当前命中切换时，编辑区滚动到目标位置；
- 当前命中在编辑器内具有明显高亮。

## 9.3 替换面板

替换区默认可折叠，展开后包含：

- 替换输入框；
- 替换当前命中按钮；
- 替换全部按钮；
- 跳过当前命中按钮；
- “保留大小写”增强选项预留。

说明：`跳过` 本质上等价于“移动到下一项但不修改内容”，但在产品语言上必须显式体现。

## 9.4 搜索结果模型

每个命中结果至少包含：

- `blockId`
- `rootId`
- `blockType`
- `blockIndex`
- `start`
- `end`
- `matchedText`
- `previewText`
- `replaceable`

## 9.5 替换执行规则

- 替换当前：只修改当前命中所在块的目标范围；
- 跳过当前：不修改数据，直接切换到下一命中；
- 全部替换：仅对当前文档、且当前规则下的全部可替换命中执行；
- 执行前后都需有明确反馈。

## 9.6 支持的块类型

V1 优先支持以下常见文本块：

- 段落 `p`
- 标题 `h`
- 列表项 `i`
- 引述中的文本叶子块
- 代码块 `c`（可配置开关）

V1 默认降级或暂不支持：

- 表格 `t`
- 数学公式 `m`
- HTML 块 `html`
- 挂件 / IFrame / 属性视图等复杂块

## 9.7 反馈与异常

插件需提供：

- 无结果提示；
- 替换成功提示；
- 全部替换结果汇总提示；
- 正则错误提示；
- 块已变化导致命中失效时的重新索引或跳过处理。

## 9.8 设置项

V1 建议提供基础设置：

- 默认是否展开替换栏；
- 是否在代码块中搜索；
- 是否显示上下文预览；
- 默认快捷键说明；
- 调试日志开关。


## 12. 技术方案约束


### 12.2 数据查询与写入

- 查询：优先通过 `/api/query/sql` 获取当前文档块集合；
- 精确内容：必要时通过 `/api/block/getBlockKramdown` 拉取块原文；
- 写入：通过 `/api/block/updateBlock` 更新块内容；
- 错误提示：通过插件消息能力统一反馈。

### 12.3 安全边界

- SQL 仅用于只读检索；
- 所有写操作走官方 API；
- 不绕过思源块模型，不直接修改底层数据库；
- 对复杂块先降级，不为了“功能全”牺牲内容安全。
